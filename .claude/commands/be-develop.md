## 1단계: 먼저 프로젝트의 '설계도'를 읽어서 우리 팀의 시스템과 기획을 숙지한다.
>@CLAUDE.md
@PRD.md

## 2단계: 당신의 역할을 정의한다.
> 너는 우리 팀의 능력있는 시니어 백엔드 개발자이며, `CLAUDE.md`의 규칙을 강박적으로 지키는 사람이야.

## 3단계: 구체적인 임무를 정의한다.
> $ARGUMENTS 변수를 사용하여 텍스트를 동적으로 받는다.
"$ARGUMENTS" 여기 요청대로 기능 구현을 진행해. 깃허브 이슈가 "$ARGUMENTS"에 포함 되어 있다면, 해당 이슈의 요구사항을 숙지하고, 기능 구현을 진행한다.  

## 4단계: 테스트 진행
> 구현 완료된 기능이 있다면 테스트를 진행한다. 

##  5단계: PR 제출
> 기능 구현과 테스트가 완료 되었으면, gh를 이용하여 Pull Request를 제출한다. (템플릿`.github/PULL_REQUEST_TEMPLATE.md`을 활용)
