## **1️⃣ 문제 정의 (Background)**

사용자는 외출 전 입고 나갈 옷을 고를 때 다음과 같은 **3단계의 복잡한 정보 탐색과 의사결정 과정**을 거칩니다.
그로 인해 준비 시간이 길어지고, 시간적 압박과 스트레스로 인한 지각으로 이어질 수 있습니다.

| **단계** | **문제 요약** | **불편 요소** |
| --- | --- | --- |
| **문제 1: 정보 탐색** | 날씨 확인 후 ‘온도별 옷차림’ 검색 | 서로 다른 앱을 오가야 함 |
| **문제 2: 옷장 매칭** | 일반 옷차림표를 내 옷과 비교 | 옷차림 표에 제시된 일반적인 옷들이 내 실제 옷의(두께감) 특징과 일치하지 않음 |
| **문제 3: 최종 결정** | 상·하의·아우터 조합 결정 | 직접 시뮬레이션해야 함 |

### 최종적인 목표

> “매일 아침 ‘오늘 뭐 입지?’를 고민하며 10분 이상 낭비되는 사용자의 의사결정 시간을 단축하는 것.”
>

---

## **2️⃣ MVP v1.0 집중 범위**

이번 MVP는 ‘문제 1: 정보 탐색의 파편화’에만 100% 집중합니다.

즉, 사용자가 **날씨 앱과 포털을 오가던 과정**을

👉 **“하나의 화면, 한 번의 호출”로 10초 이내 해결**하도록 설계합니다.

---

## **3️⃣ 핵심 가설 (Hypothesis)**

| **구분** | **내용** |
| --- | --- |
| **Given** | 사용자는 날씨 앱과 포털을 번갈아 확인하며 ‘오늘 입을 옷’을 결정한다. |
| **If** | 실시간 날씨 + 요약 코멘트 + 옷차림 카테고리를 **한 화면에서 한 번에 제공**한다면, |
| **Then** | 사용자는 여러 앱을 오가며 정보를 조합할 필요 없이 10초 만에 옷차림 방향을 잡을 수 있다. |
| **Outcome** | 정보 탐색 시간이 평균 10초 이내로 단축되고, 다음 날 재방문을 할 것이다. |

## **6️⃣ 사용자 시나리오 (User Scenario)**

| **구분** | **내용** |
| --- | --- |
| **Persona** | 김민수 (28세, 직장인, 서울 거주) |
| **Pain Point** | 날씨 앱과 ‘온도별 옷차림표’를 번갈아 보는 것이 번거롭다 |

**핵심 시나리오**

- **When:** 오전 7시 30분, 출근 준비 중
- **Context:** 평소처럼 날씨 앱을 켜려다 ‘오늘 뭐 입지?’ 앱 실행
- **Actions:**
    1. 앱 실행 즉시 “서울 15°, 맑음 ☀️ / 최고 20°, 최저 12°” 확인
    2. 바로 아래 **날씨 코멘트** 확인

       > 💬 “오전엔 쌀쌀하고 낮엔 따뜻해요. 얇은 외투를 챙기세요.”
    
    3. **시간대별 온도 변화**(현재 시간 부터 3시간 단위 날씨 변화)로 하루의 날씨를 확인

       > 07시 15°, 맑음 ☀️
       >
       >
       > 10시 17°, 흐림 🌤
       >
       > 13시 20°, 맑음 ☀️
       >
       > 16시 19°, 맑음 🌤
    
    4. **추천 옷차림 레벨별 가이드**를 보고 결정

       > **LEVEL 4 (12°C~16°C)** - 현재 온도
       > - 상의: 긴소매 티셔츠, 맨투맨, 니트/스웨터
       > - 하의: 슬랙스, 데님 팬츠
       > - 아우터: 가디건, 블레이저
       >
       > **LEVEL 5 (17°C~19°C)** - 오전/오후 예상 ⭐️ 추천
       > - 상의: 긴소매 티셔츠, 맨투맨
       > - 하의: 슬랙스, 면바지
       > - 아우터: 가디건, 바람막이
       >
       > **LEVEL 6 (20°C~22°C)** - 낮 시간대
       > - 상의: 긴소매 티셔츠, 맨투맨
       > - 하의: 슬랙스, 면바지
       > - 아우터: (불필요)

- **Result:** 날씨 확인 + 옷차림 검색이 10초 내 완료

### **Feature 2. 통합 날씨-옷차림 가이드 API**

---

| **항목** | **내용** |
| --- | --- |
| **MVP 포함 여부** | ✅ 필수 |
| **엔드포인트** | GET /api/v1/weather-outfit?city=Seoul |
| **역할** | 실시간 날씨 정보와 해당 날씨에 어울리는 옷차림을 한 번에 제공 |
| **입력값** | 사용자의 현재 위치한 도시 정보를 포함하여 요청 |
 **[처리 로직]**

(1) Geocoding API 호출
도시 → 좌표 로 변환

(2) OpenWeatherMap API 호출 → 조회 시점부터 24시간(3시간 단위)의 온도, 날씨를 조회.

(3) 오늘의 날씨 요약
→ 지금으로부터 12 시간 동안의 최대 최소 온도(일교차)와 날씨 정보를 요약

(4) 날씨 + 추천 카테고리 정보를 통합하여 JSON 응답
→ 모든 온도는 반올림하여 정수로 변환 후 레벨 매칭 (예: 11.6°C → 12°C)
→ 3에서 최대 최소 온도 범위에 포함되는 각 옷차림 레벨을 구분하여 응답한다.
→ 각 레벨(LEVEL_1~8)별로 해당하는 옷차림 카테고리(상의, 하의, 아우터, 악세서리)를 구조화하여 제공한다.
(가장 빈번한 온도의 옷차림 레벨은 별도 표시)
